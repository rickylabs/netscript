NetScript `0.0.1-beta.7` is a hotfix release for a real defect in the published `0.0.1-beta.6` line, caught in production by the `e2e-cli-prod` suite.

**The defect:** `@netscript/telemetry@0.0.1-beta.6` shipped the opt-in OpenTelemetry SDK adapter (`src/adapters/otel/otel-sdk.ts`) with statically-analyzable dynamic imports of five `@opentelemetry/*` SDK packages that are not in the package's import map. JSR rewrote those bare specifiers package-relative, so **any consumer whose module graph reached that file failed the graph-build integrity check** (`package-manifest-missing-checksum`). Concretely: `jsr:@netscript/plugin-workers@0.0.1-beta.6/services` could not start. JSR versions are immutable, so beta.6 stays broken as published — upgrade to beta.7.

**The fix (#624):** the SDK modules now load through a computed specifier that never enters the statically-analyzed module graph, preserving the zero-runtime-dependency default (OF-5). Workspace development keeps resolving through the application import scope; published consumers that opt in to the OTel SDK provider get an actionable error directing them to supply their own `SdkLoader` (the existing injection seam) backed by app-installed `@opentelemetry/*` packages.

All `@netscript/*` workspace packages are republished at `0.0.1-beta.7`; this release is otherwise identical in surface to `0.0.1-beta.6`. It also carries the production E2E hardening landed since the beta.6 cut (#613–#623: docs-only CI classification, host-agnostic agent launcher, package-source-aware prod gates, flow-b pre-warm, JSR settings reconciliation on publish).
