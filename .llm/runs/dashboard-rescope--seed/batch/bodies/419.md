## DDX-9 / S6: Run Inspector (+ NetScript run-overlay, no owned waterfall)

### Summary
The run-shaped view Aspire and Scalar have no vocabulary for: a saga instance / job attempt sequence / trigger firing / stream delivery as one logical run ‚Äî "this run is a saga on step 3 of 5, currently compensating step 2, retried once." Run-centric counterpart of S13 Live Flow (#418): S6 shows one logical run's lifecycle; S13 shows one request's causal journey ‚Äî cross-linked both ways (a run's originating request ‚Üí its S13 flow; a flow's job/saga node ‚Üí its S6 run detail).

### DX thesis
Aspire is a black-box process; Scalar is a static schema. Only NetScript knows a "run," an "attempt," a "compensation."

### Scope
- Filterable `entity-rail` run list (status + capability Selects, `empty-state` on zero-match) ‚Üí `RunDetail` (inputs/results) ‚Üí `ns-step-timeline` (marker/title/attempt-pill/duration/offset, expandable I/O CodeBlocks, All/Compact/JSON) ‚Üí `RunRail` `activity-feed` + `connector` context.
- Cross-capability `RunRecord` spanning eischat‚Üíworkers‚Üístreams as one logical run; `ExecutionRecord`, `SagaTransitionRecord`, `TriggerEvent`, stream deliveries.
- Attempt/retry vocabulary (`retrying` badge). Correlated `ns-log-stream` strip that **deep-links to Aspire logs** rather than owning them.

### Non-goals
- **NOT Aspire's Traces waterfall.** No span bars rendered here. The only trace affordance is a "View trace" out-link to Aspire `/traces/detail/{traceId}` (reverse of Aspire's "Open in Run Inspector" `withCommand`).
- The log strip does not own logs ‚Äî it deep-links Aspire structured logs.

### Acceptance criteria
- Run timeline is annotated with primitive semantics (queue name, attempt, saga step, trigger firing id), **grouped by primitive** ‚Äî not a generic span tree.
- "View trace" resolves a `traceId` via #413 and opens Aspire; no waterfall is rendered in-dashboard.
- Rerun-from-step + multi-altitude event history deferred to stable.

### Dependencies
#413 (correlation), #412 (`RunRecord`), telemetry T4‚ÄìT7 (#408) for cross-service grouping fidelity.
