# Harness Lessons

Lessons are stable cross-run observations promoted from repeated evidence. They are not a scratchpad
and they are not a second doctrine.

## Promotion Rule

Promote a lesson when:

- the same pattern appears in two or more runs, or
- one run exposes a high-impact rule that clearly applies across archetypes.

The promoting session records the source run IDs and chooses the narrowest destination.

## Destinations

Create a new file named by topic (e.g., `scope.md`, `retrieval.md`). Do not create empty placeholder
files — only create a lesson file when there is content to promote.

| Lesson type                | Example file                                |
| -------------------------- | ------------------------------------------- |
| Scope and rescope behavior | `scope.md`                                  |
| Retrieval behavior         | `retrieval.md`                              |
| Validation gotchas         | `validation.md`                             |
| Platform-specific behavior | `platform.md`                               |
| Package/plugin doctrine    | doctrine file or `netscript-doctrine` skill |
| Run mechanics              | `workflow/`                                 |

If a note is only useful for one run, keep it in that run's `worklog.md` or `context-pack.md`.
