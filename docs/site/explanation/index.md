Explanation pages are for the mental model: how NetScript is put together, why
the pieces are shaped that way, and what trade-offs come with the design.

For a practical first path, use the [tutorials](/tutorials/). To solve a
concrete task, see the [how-to guides](/how-to/). To look up exact APIs, use the
[reference](/reference/).

## Pages

- [The NetScript architecture](/explanation/architecture/) — the
  published-surface thesis, contracts-first model, the six package archetypes,
  and how plugins extend the framework.
- [The plugin model](/explanation/plugin-model/) — how plugins relate to their
  core packages, manifests and contributions, and how the host discovers them
  through generated registries.
