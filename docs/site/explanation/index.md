Explanation is **understanding-oriented**: discussion that illuminates how NetScript works and
why it is designed the way it is — architecture, plugin model, and trade-offs. It is for
reading and reflection rather than step-by-step doing. To get started practically use the
[tutorials](/tutorials/), to solve a concrete task see the [how-to guides](/how-to/), and to
look up exact APIs use the [reference](/reference/).

## Pages

- [The NetScript architecture](/explanation/architecture/) — the published-surface thesis,
  contracts-first model, the six package archetypes, and how plugins extend the framework.
- [The plugin model](/explanation/plugin-model/) — how plugins relate to their core
  packages, manifests and contributions, and how the host discovers them through generated
  registries.
