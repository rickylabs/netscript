# Support

Thanks for using NetScript. Here is where to go for each kind of help.

## Questions and usage help

- **[Discussions → Q&A](https://github.com/rickylabs/netscript/discussions/categories/q-a)** — how do
  I do X, why does Y behave this way, troubleshooting your own project.

Please do **not** open issues for usage questions — issues are for reproducible defects and tracked
work.

## Ideas and feature brainstorming

- **[Discussions → Ideas](https://github.com/rickylabs/netscript/discussions/categories/ideas)** — float
  a rough idea before it becomes a formal feature request or RFC.

## Bugs and feature requests

- **[Open an issue](https://github.com/rickylabs/netscript/issues/new/choose)** and pick the matching
  form (bug report, feature request, documentation).

## Substantial or breaking changes

- Read **[`rfcs/README.md`](rfcs/README.md)** and open an RFC. New public APIs, breaking changes, and
  cross-cutting architecture go through the RFC process.

## Security vulnerabilities

- **Do not** open a public issue. Use
  **[private vulnerability reporting](https://github.com/rickylabs/netscript/security/advisories/new)**.
  See [`SECURITY.md`](SECURITY.md).

## Release status

NetScript is pre-1.0 (`0.0.1` line). APIs may change between prereleases; breaking changes are called
out in release notes and, where substantial, driven by an RFC.
